Mosaic is an open source project that follows a transparent governance model. This document outlines how the project is currently organized and how governance may evolve as the community grows.

# Project Maintainers

The Mosaic project is initially maintained by Delos Data employees. The maintainers are responsible for:

- Reviewing and merging pull requests
- Managing releases and versioning
- Setting project direction and priorities
- Maintaining code quality and project standards
- Facilitating community discussions

# Decision Making

Currently, project decisions are made by the maintainer team through discussion in GitHub Issues and Pull Requests. For significant changes:

1. Proposals are discussed openly in GitHub Issues
2. Community feedback is welcomed and considered
3. Maintainers make final decisions based on project goals and community input

# Future Governance Evolution
As the Mosaic community grows, we intend to evolve the governance model to include non-company maintainers and more formal decision processes.

Potential future models include:

- **Community-maintainer model** - A model that includes external committers and maintainers from the broader community alongside Delos Data maintainers
- **Foundation-aligned governance** - Foundation-style governance (e.g., CNCF-style), if appropriate for the project's scale and community needs

Any governance changes will be proposed publicly and discussed in advance with the community.
We are committed to transparency and will ensure all stakeholders have an opportunity to provide input on governance evolution.
